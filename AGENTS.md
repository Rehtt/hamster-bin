# Agent Instructions

本文件记录 AI agent 在本项目中修改代码时必须遵守的额外规则。

## PROJECT.md 同步要求

每次修改代码后，必须检查是否需要更新 `PROJECT.md`。

如果本次修改影响以下任一内容，必须同步更新 `PROJECT.md`：

- 项目目录结构或新增/删除重要文件
- 关键文件职责
- 后端 API 路径、请求/响应结构或路由分组
- 数据模型、数据库迁移或持久化行为
- 配置项、环境变量或默认值
- 前端页面路由、主要页面职责或 API 客户端行为
- 运行、测试、构建或部署流程
- 静态资源嵌入方式，尤其是 `web/dist`、`embed.go`、`internal/router/router.go`
- 重要开发约束、兼容性要求或 agent 操作规则

如果代码修改不影响 `PROJECT.md` 已记录的信息，可以不更新，但最终回复中应说明未更新的原因。

## 文档维护原则

- `PROJECT.md` 面向 AI agent，优先记录项目结构、职责、约束和可执行命令。
- 保持说明简洁准确，避免写成完整用户手册或重复 README 的所有内容。
- 文档应随代码真实状态更新，不要记录尚未实现的功能作为既有事实。
- 修改文档时使用中文，保持与现有项目文档风格一致。

## 工作约束

- 修改代码前先阅读相关文件，遵循现有分层和命名风格。
- 不要覆盖用户未提交的改动；遇到无关脏工作区时只处理本次任务需要的文件。
- 优先使用现有脚本和命令验证修改。
- 最终回复中列出实际修改内容和验证结果，并明确是否已同步 `PROJECT.md`。
- 需要执行`go fix ./...`将语法迁移到最新版本。
